1.1.	Background
Школи стикаються зі зростаючими викликами у забезпеченні безпеки учнів, персоналу та відвідувачів. Традиційні методи, як охоронці чи звичайні замки, вже не забезпечують достатнього рівня захисту. Розробка сучасної системи безпеки з використанням технологій реального часу дозволить швидко реагувати на загрози, знижуючи ризики для життя і здоров'я учасників освітнього процесу.
1.2.	Business Opportunity
Ринок безпеки навчальних закладів має обмежений вибір спеціалізованих рішень. Пропонована система зосереджується на:
•	Контролі доступу,
•	Моніторингу в реальному часі,
•	Інтеграції з мобільними додатками.
Ключовою перевагою є швидке реагування та врахування специфіки освітніх закладів. Продукт може приносити прибуток через підписки, інтеграції з іншими системами та додаткові модулі.

2.1.	Vision Statement
Розроблена система спрямована на створення безпечного освітнього середовища, використовуючи сучасні технології. Мета — забезпечити:
•	Контроль доступу,
•	Постійний моніторинг,
•	Автоматичну реакцію на потенційні загрози.
Це дозволить учням вчитися у безпечному просторі, а персоналу та батькам — мати впевненість у швидкому вирішенні надзвичайних ситуацій.

2.2.	Major Features
MF-1: Моніторинг доступу
Забезпечення контролю входу та виходу в реальному часі.

MF-2: Відеоспостереження
Розміщення камер у стратегічних точках школи для віддаленого спостереження.

MF-3: Датчики диму та CO₂
Інтеграція датчиків для виявлення небезпечних ситуацій.

MF-4: Мобільний додаток
Оповіщення для батьків та персоналу.

MF-5: Тривожна кнопка
Миттєве оповіщення про загрози.

MF-6: Інтеграція з екстреними службами
Автоматичний виклик служб при серйозних інцидентах.


3.1.	Scope of Initial Release
Функціональність системи у першому випуску:
- IoT:
Камери відеоспостереження для контролю активності у школі.
Датчики руху, диму, температури, газу та вологості для моніторингу подій у реальному часі.
- Mobile:
Система сповіщень для учнів, батьків та персоналу.
Функціонал відстеження присутності учнів.
Тривожна кнопка для сигналізації екстрених ситуацій.
- Backend:
Інтеграція збору даних з камер та датчиків.
Формування базових звітів про безпеку, включаючи статистику порушень та інцидентів.
Автоматична передача сигналів до служб екстреної допомоги у разі інцидентів.
- Frontend:
Інтерфейс для адміністрації школи з можливістю перегляду записів з камер.
Відображення активності у будівлі в реальному часі.
Система сповіщень про потенційні загрози.
- Реалізація бізнес-логіки: Система повинна забезпечувати інтеграцію пристроїв (IoT) із backend для обробки даних та передачі відповідних сигналів. Для першого випуску важливо забезпечити захист персональних даних та мінімальну локалізацію функціоналу.
3.2.	Scope of Subsequent Releases
Запланована функціональність:

- IoT:
Розширення датчиків для аналізу шуму та освітлення.
Додаткові датчики для моніторингу вуличних зон.
- Mobile:
Функціонал для інтерактивного спілкування учнів, вчителів та адміністрації.
Можливість створення персональних звітів для батьків про активність учнів.
- Backend:
Система аналітики з машинним навчанням для прогнозування потенційних інцидентів.
Інтеграція з державними службами для автоматизованих звітів.
- Frontend:
Інтерактивні звіти з візуалізацією даних у вигляді графіків.
Можливість адміністрування доступу до системи через web-інтерфейс.
- У майбутніх випусках основний акцент робитиметься на розширення функціональності аналітики, інтеграції з іншими платформами безпеки та покращенні користувацького досвіду.
